Connect an MCP client

Use the hosted MCP endpoint with an OAuth-aware Streamable HTTP client. The client discovers protected-resource metadata, opens browser authorization, and stores the issued access token.

Claude Desktop / Cursor config 
{
  "mcpServers": {
    "raindrop": {
      "url": "https://raindrop-mcp.anuragd.me/mcp",
      "transport": "streamable-http"
    }
  }
}

MCP tools

The server exposes the real Raindrop tool surface. Read tools search or inspect saved content; write tools create, update, delete, bulk edit, or upload through the Raindrop API after authorization.

Collections

collection_list, collection_children_list, collection_manage, collection_bulk_delete, collection_reorder, collection_expand, collection_merge, collection_clean, collection_empty_trash, and collection_cover_upload.

Bookmarks

bookmark_search, bookmark_manage, bookmark_cache, bookmark_suggest_existing, bookmark_bulk_create, bookmark_bulk_delete, bookmark_file_upload, bookmark_cover_upload, bulk_edit_bookmarks, and bookmark_statistics.

Tags and highlights

tag_list reads the tag vocabulary, tag_manage renames, merges, or deletes tags, and highlight_manage creates, updates, deletes, or lists highlights.

User and stats

user_stats returns account-level totals, while bookmark_statistics can summarize saved content by collection, tag, type, and other Raindrop filters.

OAuth flow

Raindrop MCP uses a self-hosted authorization server in front of Raindrop.io. The MCP client gets a token scoped to the MCP resource, while the Worker stores the user Raindrop token encrypted in Cloudflare KV.

The client reads protected-resource metadata and discovers the authorization server.
The client registers dynamically, then opens browser authorization with PKCE and a resource indicator.
If no Raindrop session exists, the Worker sends the user through Raindrop login and callback.
The Worker encrypts and stores the Raindrop token, then returns the user to MCP consent.
The client exchanges the authorization code and calls the MCP endpoint with the issued bearer token.

For operator diagnostics, /info exposes the canonical URLs and tool/resource counts without exposing secrets.

Operations

The production server runs on Cloudflare Workers with static assets, Worker routes, and KV-backed OAuth storage. These public checks are safe to run without user credentials.
