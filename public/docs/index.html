Raindrop MCP Docs

Raindrop MCP exposes a Cloudflare Worker endpoint that lets MCP clients connect to Raindrop.io bookmarks through OAuth and Streamable HTTP.

Client Setup

Use the deployed MCP endpoint as the server URL in a client that supports Streamable HTTP.

https://raindrop-mcp.anuragd.me/mcp

The client should discover OAuth metadata from the well-known endpoints, complete authorization, and then call the MCP endpoint with the issued access token.

Reference Routes

  • GET /health returns service health.
  • GET /.well-known/oauth-authorization-server returns OAuth authorization server metadata.
  • GET /.well-known/oauth-protected-resource returns protected resource metadata.
  • POST /register registers dynamic OAuth clients.
  • GET /authorize and POST /token handle OAuth authorization and token exchange.
  • GET|POST|DELETE /mcp serves the MCP Streamable HTTP transport.

Operational Notes

Static assets are served from public/. Protocol routes stay worker-first in wrangler.jsonc, and missing assets should return 404 instead of falling back to a single-page app shell.
