Raindrop MCP

Connect Raindrop MCP

Use the hosted MCP endpoint with any OAuth-aware Streamable HTTP client.

{
  "mcpServers": {
    "raindrop": {
      "url": "https://raindrop-mcp.anuragd.me/mcp",
      "transport": "streamable-http"
    }
  }
}

HTTP endpoints

GET /

Landing page.

GET /docs/

This setup page.

GET /info

JSON server summary.

POST /mcp

Primary MCP Streamable HTTP endpoint.

GET /.well-known/oauth-protected-resource

MCP protected-resource metadata.

GET /.well-known/oauth-authorization-server

OAuth authorization server metadata.

GET/POST /authorize

MCP OAuth authorization and consent.

POST /token

Authorization code and refresh token exchange.

POST /register

Dynamic client registration.

GET /auth/init

Internal Raindrop login step used by /authorize.

Flow

  1. The MCP client reads protected-resource metadata from /.well-known/oauth-protected-resource.
  2. The client reads authorization server metadata from /.well-known/oauth-authorization-server.
  3. The client registers with /register.
  4. The client opens /authorize with PKCE and resource metadata.
  5. The Worker sends the user to Raindrop when no Raindrop session exists.
  6. The Worker stores the encrypted Raindrop token in KV and redirects back to /authorize.
  7. The user approves the MCP client.
  8. The client exchanges the authorization code at /token.
  9. The client calls /mcp with the issued bearer token.

Public metadata

/info returns the public server summary, available tools, resource URIs, and canonical links for clients and operators.
