How it works

From saved links to useful context. In three steps.

01

Connect Raindrop.

Your client discovers the MCP endpoint, opens OAuth, and stores the token. No pasted personal access token required.

02

Ask in your client.

Search exact pages, filter by tags, list collections, or save a new URL from the same chat where you are working.

03

Get saved context back.

The model can return your saved links, tags, highlights, and collection metadata instead of guessing from the open web.

Prompt examples

Find a saved article by memory

"I saved something about OAuth resource metadata. Find the official spec and summarize why it matters for MCP clients."

bookmark_search
{
  "search": "OAuth Protected Resource Metadata",
  "collection": 0,
  "perPage": 5
}

Save a repo while working

"Save the MCP TypeScript SDK repo, tag it mcp and sdk, and mark it important so I can find it later."

bookmark_manage
{
  "operation": "create",
  "url": "https://github.com/modelcontextprotocol/typescript-sdk",
  "tags": ["mcp", "sdk"],
  "important": true
}

Use highlights as context

"Find my saved Anthropic article and pull the highlights I left on it." Use the id returned by search in the second call.

bookmark_search
{
  "search": "Anthropic",
  "collection": 0,
  "perPage": 3

highlight_manage
{
  "operation": "list",

Add Raindrop MCP to your client

Use the MCP endpoint. Your client discovers OAuth, opens Raindrop authorization, and stores the resulting token.

Add to Claude Desktop MCP config.

{
  "mcpServers": {
    "raindrop": {
      "url": "https://raindrop-mcp.anuragd.me/mcp",
      "transport": "streamable-http"
    }
  }
}

Add to Cursor MCP config.

{
  "mcpServers": {
    "raindrop": {
      "url": "https://raindrop-mcp.anuragd.me/mcp",
      "transport": "streamable-http"
    }
  }
}

Register the hosted MCP endpoint.

codex mcp add raindrop https://raindrop-mcp.anuragd.me/mcp

Use this with any OAuth-aware Streamable HTTP MCP client.

{
  "name": "raindrop",
  "url": "https://raindrop-mcp.anuragd.me/mcp",
  "transport": "streamable-http"
}
