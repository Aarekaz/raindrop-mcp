MCP request / response

client -> mcp
{
  "method": "tools/call",
  "params": {
    "name": "bookmark_search",
    "arguments": { "search": "design" }
  }
}
mcp -> client
{
  "content": [
    { "type": "text", "text": "3 results" },
    { "type": "resource", "uri": "raindrop://item/123" }
  ]
}

What it can reach

Bookmarks Search, fetch, and reference any saved link.
Collections Navigate nested collections and their items.
Tags Filter and explore by tags you already use.
Highlights Access highlights and notes in context.

How the client sees it

ToolWhat it does
bookmark_searchSearch across saved links, titles, excerpts, tags, and collections
bookmark_manageCreate, update, move, or fetch bookmark details
collection_listList top-level and nested Raindrop collections
highlight_manageRead or write highlights attached to a bookmark

Authorization stays explicit

OAuth 2.1 with PKCE. Bookmark content remains in Raindrop and is fetched live when a tool runs.

Add Raindrop MCP to your client

Use the MCP endpoint. Your client discovers OAuth, opens Raindrop authorization, and stores the resulting token.

Add to Claude Desktop MCP config.

{
  "mcpServers": {
    "raindrop": {
      "url": "https://raindrop-mcp.anuragd.me/mcp",
      "transport": "streamable-http"
    }
  }
}

Add to Cursor MCP config.

{
  "mcpServers": {
    "raindrop": {
      "url": "https://raindrop-mcp.anuragd.me/mcp",
      "transport": "streamable-http"
    }
  }
}

Register the hosted MCP endpoint.

codex mcp add raindrop https://raindrop-mcp.anuragd.me/mcp

Use this with any OAuth-aware Streamable HTTP MCP client.

{
  "name": "raindrop",
  "url": "https://raindrop-mcp.anuragd.me/mcp",
  "transport": "streamable-http"
}
